# 📖 术语表 ｜ Glossary

> 30 条英中对照，覆盖 Dave Gauer《Ascetic Computing》全文的核心词与延伸概念。  
> 排序：按文中出现顺序 + 主题分组。

## A · 核心哲学概念

| English | 中文 | 备注 |
|---|---|---|
| Ascetic Computing | 禁欲计算 | Dave 自创的对应"aesthetic computing"的反义概念 |
| Asceticism | 禁欲主义 | 古希腊 *áskēsis* 意为"训练"，与宗教苦修无关 |
| Aesthetic Computing | 审美计算 | 为了配置美观、整齐而花时间——与禁欲相反方向 |
| Natural Asceticism | 自然禁欲 | 因追求简单而自然产生，不是为了受苦 |
| Minimalism | 极简主义 | Dave 明确说自己**不是**这个 |
| Maximalism | 多即多 | 但 Dave 是这个——"my computers are like a little electronic garden" |
| FOMO (Fear of Missing Out) | 错过焦虑 | Dave 三原则的第二条 |
| Shiny Things | 闪亮新玩具 | Dave 三原则的第三条，也是他自己承认的"final boss" |
| Deprivation | 剥夺 / 自我惩罚 | Dave 明确说这**不是**目标，只是禁欲的副作用 |
| Faustian Bargain | 浮士德交易 | 拿原则换便利的妥协，Dave 坦承自己有几个 |

## B · 思维框架 / 经济学

| English | 中文 | 备注 |
|---|---|---|
| Lindy Effect | Lindy 效应 | "一个东西活得越久，预期寿命也越长"——Taleb 推广 |
| Constraint = Creativity | 约束等于创造力 | Dave 引《Programming Pearls》第一章 |
| Programming Pearls | 编程珠玑 | Jon Bentley 1986 年经典，"位图排序"案例 |
| Transient Knowledge | 短暂知识 | BIOS 设置、专有快捷键，单次使用，难迁移 |
| Lasting Knowledge | 持久知识 | Unix 基础、编程基础、类型论 |
| Use Something in Anger | 动真格地用 | 英语习语，指生产现场冒冷汗时刻 |
| Going Inward | 向内走 | Dave 全文最后一句——离线即向内 |
| Be Boring | 做个无聊的人 | Austin Kleon 书中章节，Dave 引为禁欲注脚 |
| Flaubert 名言 | "Be regular and orderly in your life, so that you may be violent and original in your work." | 全文精神锚点之一 |

## E · 文中近义但不可混淆

| 看似一样，其实不一样 |
|---|
| **Minimalism vs Asceticism**：前者关心"少"，后者关心"原则" |
| **Aesthetic vs Ascetic**：前者关心"美观"，后者关心"省略" |
| **Deprivation vs Restraint**：前者是苦行，后者是克制 |
| **Lindy vs Conservative**：前者是基于寿命的预测，后者是情绪上的保守 |
| **Maximalist vs Hoarder**：前者是同类多份，后者是杂物堆积 |

## F · 中文圈对应概念

| English | 中文圈类似概念 | 备注 |
|---|---|---|
| Ascetic Computing | "极简主义工程" / "工具节制" | 但不完全等同——禁欲比极简更强调"原则" |
| FOMO | 错过焦虑 / 比较焦虑 | 中文圈常说"卷"，包含 FOMO 但不限于 |
| Lindy Effect | "经典即可靠" / "老办法稳" | 工程师常说"无聊技术堆栈"，与 Lindy 同向 |
| Faustian Bargain | "灵肉相易" / 浮士德式契约 | 哲学文献中已有翻译 |
| Going Inward | "向内观照" / "向内求" | 与禅宗"内观"有部分重合 |
